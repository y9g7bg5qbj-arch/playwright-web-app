# Vero IDE - UI/UX Design Brief

## Product Overview

**Vero IDE** is a test automation platform designed for non-technical QA testers. Think of it as a modern code editor (like VS Code, Cursor, or Windsurf) specifically built for creating, running, and managing automated browser tests.

**Target Users**: QA testers who may not be developers but need to create automated tests
**Competitor Reference**: LeapWork, testRigor, Playwright Codegen
**Design Inspiration**: Cursor IDE, Windsurf IDE, VS Code, IntelliJ IDEA, Zed Editor

---

## Design Philosophy

### Core Principles
1. **Minimal Clicks** - Users should complete tasks in 3-4 clicks maximum
2. **Maximum Context** - Show relevant information without navigating away
3. **AI-First** - AI assistance is integral, not an afterthought
4. **Progressive Disclosure** - Show complexity only when needed (expandable sections)
5. **Non-Technical Friendly** - Avoid developer jargon, use clear labels

### Visual Direction
- **Theme**: Dark mode (GitHub's dark palette)
- **Aesthetic**: Clean, professional, modern IDE look
- **Typography**: Sans-serif for UI (Inter), Monospace for code (Fira Code)
- **Density**: Comfortable spacing, not cramped

## Animation Guidelines

- **Panel transitions**: 200ms ease-out
- **Hover states**: 150ms ease
- **Expandable sections**: 200ms with height animation
